# 2026-03-07 技術重點紀錄 (NOTES.md)

## 1. 非阻塞 HTTP 請求的權衡
在語音代理場景中，偵測到喚醒詞後的每一毫秒都很珍貴。雖然 `esp_http_client_perform` 是同步阻塞的，但透過將 `timeout_ms` 設定為 1000ms，我們能確保就算伺服器反應稍慢，也不會嚴重耽誤隨後的錄音任務。

## 2. 跨協議聯動 (HTTP + WebSocket)
雖然主要的事件流走 WebSocket，但針對這種單向、立即性的信號觸發（如播放特定的 ACK 音效），直接使用 HTTP GET 是一個輕量且解耦的好選擇。

## 3. 未來優化方向
若未來發現 1s 阻塞仍感明顯，可考慮將 HTTP 請求放入獨立的 FreeRTOS Task 中執行。
