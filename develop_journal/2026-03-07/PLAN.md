# 2026-03-07 喚醒提示音優化計畫 (PLAN.md)

## 1. 背景與需求
目前使用者對著 ESP32 說出喚醒詞後，雖然有 LED 閃爍，但缺乏即時的語音反饋（音效）。為了提升使用者體驗，需要在偵測到喚醒詞時，讓伺服器立即播放提示音。

## 2. 實作方案
- **ESP32 端**: 在辨識成功後，立即發送一個非阻塞的 HTTP GET 請求到伺服器的 `/ack` 路由。
- **Server 端**: 已實作 `/ack` 路由，負責觸發本地音效播放。
- **效能考量**: HTTP 請求需設定極短的逾時（1s），避免阻塞後續的 3 秒語音錄製。

## 3. 預期效益
提供即時的語音反饋，讓使用者確認系統已進入「聆聽」狀態。
